# План инкрементов CJXplorer

## Принцип

Каждый инкремент — самостоятельная работающая единица, которую можно показать и получить обратную связь. Следующий инкремент расширяет предыдущий, не ломая его.

---

## Инкремент 1: Telegram + Evaluation Agent

**Цель:** валидировать, что LLM способна оценивать CJ по критериальной модели.

**Что делает:**
- Telegram-бот принимает набор скриншотов
- Evaluation Agent (LLM + системный промпт с критериями) анализирует их
- Результат возвращается в Telegram в текстовом виде

**Стек:** Python, python-telegram-bot, OpenAI/Claude API

**Блоки на схеме:** Telegram Bot → Orchestrator (простой) → Evaluation Agent → Telegram Bot

**Результат:** работающий бот, который оценивает CJ по фото. Можно тестировать качество промпта и критериальной модели.

---

## Инкремент 2: Comparison Agent

**Цель:** сравнение нескольких CJ между собой.

**Что добавляет:**
- Пользователь загружает скриншоты нескольких приложений (помечая какое есть какое)
- Comparison Agent формирует сравнительную таблицу по критериям
- Итоговый вердикт: кто лучше/хуже и по каким параметрам

**Блоки:** + Comparison Agent

---

## Инкремент 3: Report Generator + хранение

**Цель:** сохранять результаты и генерировать полноценные отчёты.

**Что добавляет:**
- Media Storage (S3 или локальная папка) для скриншотов
- Report Storage (DB) для истории оценок
- Report Generator: выдача в PDF / HTML / MD
- Начало трекинга динамики (было 🔴 → стало 🟡)

**Блоки:** + Media Storage, Report Storage, Report Generator

---

## Инкремент 4: Design Agent (Figma/Pixso)

**Цель:** автоматически забирать экраны макетов по ссылке.

**Что добавляет:**
- Подключение к Figma API и Pixso API
- Пользователь даёт ссылку → агент сам вытягивает все экраны
- Экраны уходят в Evaluation Agent как обычные скриншоты

**Блоки:** + Design Agent

---

## Инкремент 5: Navigation Agent Web (Playwright)

**Цель:** автономное прохождение CJ на сайтах.

**Что добавляет:**
- Navigation Agent Web управляет браузером через Playwright
- Делает скриншоты на каждом шаге
- Фиксирует время, ошибки, редиректы
- Передаёт результаты в Evaluation Agent

**Блоки:** + Navigation Agent Web, Target websites

---

## Инкремент 6: Navigation Agent Mobile (OpenClaw)

**Цель:** автономное прохождение CJ в мобильных приложениях.

**Что добавляет:**
- OpenClaw Gateway + iOS/Android Nodes
- Navigation Agent iOS и Android — управляют устройствами через REST API
- Скриншоты через canvas.snapshot
- Результаты → Evaluation Agent

**Блоки:** + Nav Agent iOS, Nav Agent Android, OpenClaw Gateway, Nodes

---

## Инкремент 7: Dashboard + Web UI

**Цель:** полноценный интерфейс вместо только Telegram.

**Что добавляет:**
- Web UI для загрузки скриншотов, ввода ссылок, запуска исследований
- Dashboard с историей оценок, сравнениями, динамикой
- Knowledge Base (VectorDB) для хранения критериев и эвристик

**Блоки:** + Web UI, Dashboard, Knowledge Base (VectorDB)

---

## Инкремент 8: Внутренний контур + Hadoop

**Цель:** обогащение оценок внутренними данными банка.

**Что добавляет:**
- File Gateway для обмена между контурами
- Internal Evaluation Agent на внутренней LLM
- Подключение к витринам: ClickStream, Обращения, NPS/CSI
- Internal Dashboard

**Блоки:** + File Gateway, Internal Eval Agent, Internal Mockup Analyzer, витрины, Internal Dashboard

---

## Инкремент 9: Петля обучения

**Цель:** повышение качества оценок через верификацию.

**Что добавляет:**
- Интерфейс для экспертной верификации (agree/disagree с оценкой агента)
- Learning Base — накопление верифицированных примеров
- Обновление промптов / few-shot примеров на основе обратной связи
- В перспективе: fine-tuning модели

**Блоки:** + Human Verification, Learning Base, Model Improvement

---
