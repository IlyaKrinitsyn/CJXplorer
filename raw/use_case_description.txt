Работа с ИИ-агентом по анализу Customer Journey (CJ)
 
1. ЦЕЛЬ
 
Автоматизировать анализ клиентского пути, выявлять узкие места, давать рекомендации по улучшению, сравнивать с конкурентами, присваивать степень критичности (🔴 / 🟡 / 🟢) и выдавать итоговое заключение.
 
2. ТЕКУЩИЙ ПРОЦЕСС (ВЕРСИЯ 1.0)
 
Этап 1 — Загрузка данных
 
Любой сотрудник продукта или подразделения загружает в агента:
- CJ-схему / карту пути
- Тексты коммуникаций (email, push, смс, скрипты)
- Скриншоты экранов (сайт, приложение, чекаут)
 
Этап 2 — Анализ
 
Агент выполняет:
1. Распознавание и структурирование шагов CJ
2. Оценку сквозной логики (поиск разрывов и противоречий)
3. Сравнение с конкурентами (бенчмарк по этапам)
4. Присвоение критичности каждому недостатку: 🔴 / 🟡 / 🟢
 
Этап 3 — Выдача заключения
 
Агент формирует итоговое заключение на основе проведенного анализа.
 
Этап 4 — Действия пользователя
 
- 🔴 Красный → запуск запрещен, требуются правки
- 🟡 Желтый → запуск возможен с планом доработок
- 🟢 Зеленый → запуск разрешен
 
 
3. БУДУЩЕЕ РАЗВИТИЕ (ВЕРСИЯ 2.0)
 
Агент научится **самостоятельно заходить в мобильное приложение** и проводить авто-тест клиентского пути:
 
- Открывать приложение
- Проходить регистрацию / авторизацию
- Добавлять товар в корзину
- Оформлять заказ
- Фиксировать время, ошибки, зависания, расхождения с ожидаемым сценарием
 
После завершения теста агент **выдает заключение** (аналогично версии 1.0) и при необходимости прикладывает скриншоты с проблемных шагов.
